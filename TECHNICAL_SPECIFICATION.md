# Техническое задание: ZapretGUI Mod

## 1. Введение

### 1.1. Назначение документа
Данный документ описывает технические требования и спецификацию разработки графической оболочки для утилиты zapret с функциями оптимизации сетевого подключения для операционной системы Windows.

### 1.2. Область применения
Документ предназначен для разработчиков, тестировщиков и технических специалистов, участвующих в создании и поддержке приложения ZapretGUI.

### 1.3. Термины и определения

| Термин | Определение |
|--------|-------------|
| DPI | Deep Packet Inspection — технология анализа сетевого трафика |
| Zapret | Утилита для обхода DPI-блокировок |
| TCP | Transmission Control Protocol — протокол управления передачей |
| DNS | Domain Name System — система доменных имён |
| MTU | Maximum Transmission Unit — максимальный размер пакета |
| BBR | Bottleneck Bandwidth and RTT — алгоритм контроля перегрузок |
| ECN | Explicit Congestion Notification — явное уведомление о перегрузке |

## 2. Общее описание

### 2.1. Назначение продукта
ZapretGUI — это Windows-приложение с графическим интерфейсом, предназначенное для:
- Управления утилитой zapret (обход DPI-блокировок)
- Оптимизации сетевого стека Windows
- Упрощения настройки обхода блокировок для конечных пользователей

### 2.2. Целевая аудитория
- Пользователи Windows 10/11, желающие обойти блокировки сервисов
- Пользователи, не имеющие опыта работы с командной строкой
- Геймеры, нуждающиеся в оптимизации сетевого подключения

### 2.3. Предположения и зависимости
- Требуется наличие прав администратора для применения системных настроек
- Требуется .NET 8 Desktop Runtime
- Утилита zapret должна быть доступна отдельно или в комплекте

## 3. Функциональные требования

### 3.1. Управление zapret

#### 3.1.1. Запуск и остановка
- **FR-001**: Приложение должно обеспечивать запуск процесса zapret с заданными параметрами
- **FR-002**: Приложение должно обеспечивать корректную остановку процесса zapret
- **FR-003**: Приложение должно отслеживать статус процесса (работает/остановлен)
- **FR-004**: При аварийном завершении процесса должно выводиться уведомление

#### 3.1.2. Конфигурация параметров
- **FR-005**: Поддержка выбора метода обхода (fake, http, tls, mix)
- **FR-006**: Возможность указания списка доменов для обхода
- **FR-007**: Возможность указания списка IP-адресов/подсетей
- **FR-008**: Настройка портов (по умолчанию 80, 443)
- **FR-009**: Выбор сетевого интерфейса (авто или ручной)
- **FR-010**: Поддержка дополнительных аргументов командной строки

#### 3.1.3. Профили
- **FR-011**: Встроенные профили: Discord, YouTube, Telegram, All Services, Custom
- **FR-012**: Возможность создания пользовательских профилей
- **FR-013**: Сохранение профилей в формате JSON
- **FR-014**: Экспорт и импорт профилей

### 3.2. Оптимизация сети

#### 3.2.1. TCP оптимизация
- **FR-015**: Настройка TCP Auto-Tuning Level
- **FR-016**: Включение/отключение ECN
- **FR-017**: Включение/отключение TCP Timestamps
- **FR-018**: Включение/отключение Selective ACK
- **FR-019**: Отключение алгоритма Nagle для выбранных приложений
- **FR-020**: Включение BBR/CTCP (при поддержке ОС)
- **FR-021**: Сброс TCP настроек к значениям по умолчанию

#### 3.2.2. DNS оптимизация
- **FR-022**: Установка пользовательских DNS-серверов
- **FR-023**: Предустановленные DNS (Cloudflare, Google, Quad9)
- **FR-024**: Очистка DNS-кэша (ipconfig /flushdns)
- **FR-025**: Сброс DNS к DHCP
- **FR-026**: Отображение текущих DNS-серверов

#### 3.2.3. MTU оптимизация
- **FR-027**: Отображение текущего MTU для интерфейсов
- **FR-028**: Автоматический подбор оптимального MTU (ping-тесты)
- **FR-029**: Ручная установка MTU
- **FR-030**: Сброс MTU к значению по умолчанию (1500)

### 3.3. Системные функции

#### 3.3.1. Автозагрузка
- **FR-031**: Добавление приложения в автозагрузку через Task Scheduler
- **FR-032**: Удаление из автозагрузки
- **FR-033**: Настройка задержки запуска (5 секунд)

#### 3.3.2. Системный трей
- **FR-034**: Отображение иконки в области уведомлений
- **FR-035**: Контекстное меню: Вкл/Выкл, Смена профиля, Показать, Выход
- **FR-036**: Сворачивание в трей при закрытии окна
- **FR-037**: Двойной клик по иконке открывает окно

#### 3.3.3. Резервное копирование
- **FR-038**: Создание точки восстановления Windows перед изменениями
- **FR-039**: Экспорт настроек реестра в .reg файлы
- **FR-040**: Импорт настроек из резервной копии
- **FR-041**: Хранение резервных копий в %APPDATA%\ZapretGUI\Backups

### 3.4. Логирование

#### 3.4.1. Логи zapret
- **FR-042**: Перехват stdout/stderr процесса zapret
- **FR-043**: Отображение логов в реальном времени в GUI
- **FR-044**: Сохранение логов в файл
- **FR-045**: Очистка лога

#### 3.4.2. Логи приложения
- **FR-046**: Ведение лога работы приложения (Serilog)
- **FR-047**: Ротация логов (7 дней)
- **FR-048**: Путь: %APPDATA%\ZapretGUI\Logs\

## 5. Технические требования

### 5.1. Платформа
- **TECH-001**: ОС: Windows 10 x64 (версия 1903+) или Windows 11
- **TECH-002**: Фреймворк: .NET 8 Desktop Runtime
- **TECH-003**: Язык: C# 12
- **TECH-004**: UI фреймворк: Windows Forms

### 5.2. Зависимости

| Пакет | Версия | Назначение |

### 5.3. Требования к производительности
- **PERF-001**: Потребление ОЗУ: не более 50 МБ в простое
- **PERF-002**: Время запуска приложения: не более 3 секунд
- **PERF-003**: Запуск zapret: не более 1 секунды
- **PERF-004**: Применение оптимизаций: не более 30 секунд

### 5.4. Требования к безопасности
- **SEC-001**: Запуск с правами администратора (manifest requireAdministrator)
- **SEC-002**: Валидация всех пользовательских входных данных
- **SEC-003**: Защита от инъекций команд (экранирование аргументов)
- **SEC-004**: Безопасная работа с реестром (проверка путей)
- **SEC-005**: Отсутствие хранения чувствительных данных

### 5.5. Требования к надёжности
- **REL-001**: Обработка всех исключений с логированием
- **REL-002**: Гарантия отката изменений при ошибке
- **REL-003**: Контроль единственного экземпляра (Mutex)
- **REL-004**: Корректное освобождение ресурсов (IDisposable)
- **REL-005**: Восстановление после сбоя питания (точки восстановления)

## 10. Приложения

### Приложение A: Список поддерживаемых команд zapret

```
--preset=<name>          Предустановленный профиль
--method=<method>        Метод обхода (fake, http, tls)
--dpi-list=<file>        Файл со списком доменов
--ip-list=<file>         Файл со списком IP
--ports=<ports>          Список портов
--iface=<interface>      Сетевой интерфейс
--dpi-desync=<method>    Метод десинхронизации
--host-mismatch=<0|1>    Несоответствие хоста
```
